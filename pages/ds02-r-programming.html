5.3 Important Attributes for read.table

  • file, name of a file or a connection
  • header, logical indicating if the file has a header line
  • sep, a string indicating how the columns are seperated
  • colClasses, a character vector indicating the class of each column in the dataset
  • nrows, number of rows in the dataset
  • comment.char, a characger string indicating the comment character
  • skip, the number of lines to skip from the beginning
  • stringsAsFactors, should character variables be coded as factors?

5.4 Reading in Larger Datasets with read.table

  • Set comment.char = "" if there are no commented lines in your file (for speeding up reading)
  • If all of the columns are "numeric", for example, then you can just set colClasses = "numeric"
  • A quick, dirty way to figure out the classes of each column is the following: 
    initial <- read.table("datatable.txt", nrows = 100)
classes <- sapply(initial, class)
tabAll <- read.table("datatable.txt", colClasses = classes)
  • Set nrows. This doesn't make R run faster but it helps with memory usage (you can use the Unix tool wc to calculate the number of lines in a file)
  • Calculating Memory Requirements (eg. 1,500,000 rows, 120 columns):
    1,500,000 x 120 x 8 bytes/numeric
    = 1,440,000,000 bytes
    = 1,440,000,000 / 2^20 bytes/MB
    = 1,373.29 MB
    = 1.34 GB

6 Subsetting

  • [, select more than one element (returns an object of the same class as the original)
  • [[, select a single element (extract elements of a list or a data frame)
  • $, used to extract elements of a list or data frame by name
y[y > 0]                # get all positive elements of y
x[!is.na(x) & x > 0]    # remove all NAs and get all positive elements

x[c(3,5,7)]             # subsetting elements 3, 5, 7
x[c(-2, -10)]           # gives us all elements of x EXCEPT for the 2nd and 10 elements
x[-c(2, 10)]            # same as above

x[c("foo","bar")]       # return elements of named vector

6.1 Subsetting Lists

x <- list(foo = 1:4, bar = 0.6)

# [1] 0.6

# $bar
# [1] 0.6
x <- list(foo = 1:4, bar = 0.6, baz = "hello")

x$name ## element name doesnt exist!

  • An environment in R is a collection of symbol/value pairs

  • Lexical Scoping:

    make.power <- function(n) {
    pow <- function(x) {
        x^n     # n is a free variable
    }
    pow
}

cube <- make.power(3)
square <- make.power (2)

cube(3)     # returns 27
square(3)   # returns 9

# What's in a function's environment
ls(environment(cube))         # returns "n" and "pow"
get("n", environment(cube))   # returns 3

  • Lexical vs Dynamic Scoping:

    y <- 10

f <- function(x) {
    y <- 2
    y^2 + g(x)
}

g <- function(x) {
    x*y
}

# What is the value of:
f(3)
    • With Lexical Scoping the value of y in the function g is looked up in the environment in which the function was defined, in this case the global environment, so the value of y is 10
    • With Dynamic Scoping the value of y is looked up in the environment from the function was called (sometimes referred to as the calling environment; in R the calling environment is known as the parent frame), so the value of y is 2

11 Loop functions

  • lapply: Loop over a list and evaluate a function on each element
    • An auxiliary function split is useful, particulary in conjunction with lapply
  • sapply: Same as lapply but try to simplify the result
  • vapply: Same as sapply but allows you to specify the correct format of the result explicitly
  • apply: Apply a function over the margins of an array
  • tapply: Apply a function over subsets of a vector
  • mapply: Multivariate version of lapply

11.1 lapply

  • lapply takes three arguments: (1) a list X; (2) a function FUN; (3) other arguments via its ... argument
x <- list(a = 1:5, b = rnorm(10))
lapply(x, mean)
x <- 1:4
lapply(x, runif, min = 0, max = 10)
x <- list(a = matrix(1:4, 2, 2), b = matrix(1:6, 3, 2))
lapply(x, function(elt) elt[,1])

11.2 apply

  • apply is used to a evaluate a function (often an anonymous one) over the margins of an array
x <- matrix(rnorm(200), 20, 10)
colMeans = apply(x, 2, mean)               # take mean for each column
rowSums = apply(x, 1, sum)                 # take sum for each row
x <- matrix(rnorm(200), 20, 10)
apply(x, 1, quantile, probs = c(0.25, 0.75))
a <- array(rnorm(2 * 2 * 10), c(2, 2, 10))
apply(a, c(1, 2), mean)
rowMeans(a, dims = 2)                      # same as above

11.3 mapply

  • mapply is a multivariate apply of sorts which applies a function in parallel over a set of arguments
list(rep(1, 4), rep(2, 3), rep(3, 2), rep(4, 1))
mapply(rep, 1:4, 4:1)                               # same as above
noise <- function(n, mean, sd) {
    rnorm(n, mean, sd)
}

mapply(noise, 1:5, 1:5, 2)
# same as above
list(noise(1, 1, 2), noise(2, 2, 2),
    noise(3, 3, 2), noise(4, 4, 2),
    noise(5, 5, 2))

11.4 tapply

  • tapply is used to apply a function over subsets of a vector

13.1.2 Generating Random Numbers From a Linear Model

  • Suppose we want to simulate from the following linear model

    $y = \beta_0 + \beta_1 + \epsilon$

    where $\epsilon \sim N(0,2^2)$. Assume $x \sim N(0,1^2)$, $\beta_0 = 0.5$ and $\beta_1 = 2$

    set.seed(20)
x <- rnorm(100)
e <- rnorm(100, 0, 2)
y <- 0.5 + 2 * x + e

summary(y)
plot(x, y)

  • What if $x$ is binary?

    set.seed(20)
x <- rbinom(100, 1, 0.5)
e <- rnorm(100, 0, 2)
y <- 0.5 + 2 * x + e

summary(y)
plot(x, y)

  • Suppose we want to simulate from a Poisson model where

    Y $\sim$ Poisson$(\mu)$

    log $\mu = \beta_0 + \beta_1 x$

    and $\beta_0 = 0.5$ and $\beta_1 = 0.3$

    set.seed(1)
x <- rnorm(100)
log.mu <- 0.5 + 0.3 * x
y <- rpois(100, exp(log.mu))        # take exp() of log.mu to get the mean of a poisson distribution

summary(y)
plot(x, y)

15 Optimizing Code

15.1 Using system.time

  • Returns an object of class proc_time
    • user time: time charged to the CPU(s) for this expression
    • elapsed time: "wall clock" time
  • Usually, the user time and elapsed time are relatively close, for straight computing tasks
  • Elapsed time may be greater than user time if the CPU spends a lot of time waiting around
  • Elapsed time may be smaller than the user time if your machine has multiple cores/processors
    (and is capable of using them)
    • Multi-threaded BLAS libraries (vecLib/Accelerate, ATLAS, ACML, MKL)
    • Parallel processing via the parallel package
  • Timing Longer Expressions (use curly brackets) 
    system.time({
    n <- 1000
    r <- numeric(n)
    for (i in 1:n) {
        x <- rnorm(n)
        r[i] <- mean(x)
    }
})

15.2 R Profiler Rprof

  • The Rprof() function starts the profiler
  • The summaryRprof() function summarizes the output from Rprof() (otherwise its not readable)
  • DO NOT use system.time() and Rprof() together or you will be sad
  • There are two methods for normalizing the data
    • "by.total" divides the time spend in each function by the total run time
    • "by.self" does the same but first subtracts out time spent in functions above in the call stack
  • C or Fortran code is not profiled

    • What are the variable names?
    • How are the variables stored?
    • Are there missing data?
    • Are there any flaws in the data?
  • Useful functions:
    1. ls(), list the variables in your workspace
    2. class(data). give us a clue as to the overall structure of the data
    3. dim(data), see exactly how many rows and columns we're dealing with
      • nrow(data), just to see how many observations
      • ncol(data), to see the number of columns
    4. object.size(data), how much space is the dataset occupying in memory
    5. names(data), returns a character vector of column (ie. variable) names
      • head(data, 10), allows to preview the the top of the dataset
      • tail(data, 10), allows to preview the end of the dataset
    6. summary(data), to get a better feel for how each variable is distributed and how much of the dataset is missing
      • table(data$factor_name), to see how many times each value occurs in the data within a category/factor
    7. str(data), summary of the above, ie. class, obs & cols, name & class of each variable, preview of its contents
